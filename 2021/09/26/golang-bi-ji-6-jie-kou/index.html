Golang笔记(6)--接口

无标签
发布日期:   2021-09-26
文章字数:   1.5k

接口是Go语言的设计之道，它的核心思想在于接口和实现者的解耦，接口实现者只需知道实现的是什么样子的接口，但无须指明实现哪一个接口。编译器知道最终编译时使用哪个类型实现哪个接口，或者接口应该由谁来实现。有一种说法更为贴切，称接口为Protocol，即接口认为是双方都接受的一种合作协议，设计接口的不必过问使用者会在什么条件下使用，使用者也不必了解接口实现的细节。

接口的定义

接口形式代码：

type 接口类型名 interface {
    方法名1( 参数列表1 ) 返回值列表1
    方法名2( 参数列表2 ) 返回值列表2
    ...
}
  • 接口类型名：使用type将接口定义为自定义的类型名，一般结尾命名为er
  • 方法名：方法名和接口名的首字母均为大写时，可以被接口所在包之外的代码访问
  • 参数列表、返回值列表：与函数或方法的定义方式类似，参数变量名可以省略

Go语言中的接口简而精，精准描述本接口的功能，就可以使用多个简单的接口嵌入或组合为复杂接口。

接口的实现

  • 条件一：接口的方法与实现接口的类型方法格式一致

格式包括函数名和方法签名，实现内容必须一致，下面是对接口实现的一个简单示例：

package main

import (
	"fmt"
)

type DataWriter interface {
    WriteData(data interface{}) error
}

type file struct {
}

//file这个结构实现了WriteData接口
//这里面方法名必须和接口定义的名字一致，里面的签名(方法)也应该保持一致
//比如把WriteData写成writedata，把interface{}写成string，都是不允许的，会在29行报错
func (d *file) WriteData(data interface{}) error {
    fmt.Println("WriteData:", data)
    return nil
}

func main() {
    
    f := new(file)
    
    var writer DataWriter
    
    //f之所以能赋值给writer，就是因为它实现了writer接口中的WriteData()方法
    writer = f
    
    writer.WriteData("data")
}
  • 条件二：接口定义的所有方法都要被实现

上面的例子中，接口只定义了一个方法，但当一个接口中有多个方法时，只有这些方法都被实现了，接口才能被正确编译并使用。

那么在类似于Java的SpringBoot架构中，经常会看到一些项目中的类，是一层一层派生的，然后可以构造出一棵非常复杂的派生树，这是因为这些语言强耦合的构建关系，但如之前所言，Go语言是弱耦合的，所以在Go中不会出现这些复杂的派生树。对于 Go 语言来说，非侵入式设计让实现者的所有类型均是平行的、组合的。使用 GO 语言时，不需要同时也不可能有“类派生图”，开发者唯一需要关注的就是“我需要什么？”，以及“我能实现什么？”。

语言类型和接口的关系

  • 一个类型可以实现多个接口

这个意思就是说，比如我定义了一个结构体，这个结构体编写了若干个回调函数，这些方法中既包含了接口A，也包含了接口B、C……举例如下：

//这里定义了一个写的接口A
type Writer interface {
    Write(p []byte) (n int, err error)
}

//这里定义了一个关闭接口B
type Closer interface {
    Close() error
}

//定义一个网络连接的结构体
//当然，接口的实现并不局限于结构体，任何类型都可以实现接口
type Socket struct {
}

//这个结构体类型实现写接口A
func (s *Socket) Write(p []byte) (n int, err error) {
    return 0, nil
}

//这个结构一类型实现关闭接口B
func (s *Socket) Close() error {
	return nil
}
  • 多个类型可以实现相同接口

一个接口的方法，不一定需要由一个类型完全实现，接口的方法可以通过在类型中嵌入其他类型或者结构体来实现。也就是说，使用者并不关心某个接口的方法是通过一个类型完全实现的，还是通过多个结构嵌入到一个结构体中拼凑起来共同实现的。

接口的范例——sort.Interface接口

对于常见的数据类型，接口已经提供了轮子，直接使用即可，对于想要自己实现这个接口，则必须自行实现接口的三个方法：

type Interface interface {
    // 获取元素数量
    Len() int
    
    // 小于比较
    Less(i, j int) bool
    
    // 交换元素
    Swap(i, j int)
}

标准库在sort包中提供了sort.Slice()函数进行了更简单的排序，这里面需要传入待排序的数据，以及实现less的方法定义，函数定义如下：

func Slice(x interface{}, less func(i, j int) bool)

下面是一个实现的实例，仅体现部分代码：

sort.Slice(heros, func(i, j int) bool {
        if heros[i].Kind != heros[j].Kind {
            return heros[i].Kind < heros[j].Kind
        }
        return heros[i].Name < heros[j].Name
    })

它就是直接在这个方法里面再把这个less方法给实现掉就完事了。
